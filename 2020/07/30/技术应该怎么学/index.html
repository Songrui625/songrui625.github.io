技术应该怎么学

你是否喜欢遇到一门新技术就先去买一本书、就去找视频。这篇文章想介绍下怎么学习一门快速了解并学习一门新的技术。

一门新技术是怎么推广的？

在日常生活中，一般我们买了一个新的产品，是怎么学会使用的？可能大家都会想到，买产品它送了个使用手册(当然大多数情况还是会请师傅帮忙)。有了使用手册我们就可以按照他上面的指南进行操作。

同理，当一门新的技术出现之后，它为了吸引用户，必然需要推广，为了让用户知道如何使用，那肯定要有良好的用户文档。

用户文档一般分为两种：参考文档(Reference Doc) 和API文档(API Doc)

如果参考文档写得不好，用户觉得难以使用，那用户将会放弃这个产品。所以市面上优秀的产品一定是有良好的用户文档的，比如Alibaba的分布式RPC框架Dubbo、前端UI框架Bootstrap、前端Js框架Vue和对Java影响深远的Spring框架。

就我个人而言，学习一门新的技术我首先会去去看这门技术的官网，看一下这门技术的介绍，结构，解决了什么难点。然后通过参考文档的快速使用，自己编写一个简单的demo。然后通过文档建立知识结构。遇到使用的难点会去github官网issue看下有没有解答。最后才是通过书籍、视频和技术文档补充不会的。

通过文档学习

以Spring Framework为例子，讲述一下如何通过文档学习技术。
打开Spring的官网,在Projects栏中中可以打开Spring中任何一个项目介绍，打开Spring Framework：

可以看到有Reference DOC 和 API DOC两个选项，前面的数字属于版本号。

GA表示正式发行版、CURRENT表示当前最新版本、SNAPSHOT表示简要。

我们打开当前的正式发行版的Reference Doc，选择核心（core）章节。

我们可以通过侧边栏的目录进行知识点的构建，对每一个知识点进行学习，同时一定要自己用代码去实现。技术一定是要实践练习才能掌握的。
